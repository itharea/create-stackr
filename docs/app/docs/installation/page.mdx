# Installation

stackr is run through `npx` — there is nothing to install globally. This page
covers the prerequisites for generating a project and booting it locally.

## Prerequisites

<Table
    headers={['Requirement', 'Version', 'Needed for']}
    rows={[
        ['Node.js', '22 or newer', 'Running the CLI and every service'],
        ['A package manager', 'npm, yarn, or bun', 'Installing workspace dependencies'],
        ['Docker', 'Recent', 'docker:dev and the test profiles'],
        ['Git', 'Any', 'Version control (recommended)'],
    ]}
/>

<Callout type="info" title="Docker is for running, not generating">
    The CLI itself does not require Docker. You only need it to bring the stack up
    with <code>docker:dev</code> or to run the Docker-based test profiles. If you
    prefer, you can run a single service against a host-mode database instead.
</Callout>

### Node.js

stackr targets **Node.js 22+** (its `engines.node` floor), and the generated
services are tested on the current LTS. Check your version:

<CodeBlock code="node --version" language="bash" />

### Package manager

npm, yarn, and bun are all first-class — you pick one during generation and every
generated script branches on that choice. **bun** is the recommended default for
its speed, but nothing in the generated project is bun-only.

## Generating a project

Run the CLI with `npx`; it always uses the latest published version:

<CodeBlock code="npx create-stackr@latest my-app" language="bash" />

This launches the interactive wizard. To skip the prompts and generate the default
shape (Drizzle, an auth service with admin dashboard, and one `core` base service),
pass `--defaults`:

<CodeBlock code="npx create-stackr@latest my-app --defaults" language="bash" />

<Callout type="info" title="Two binaries">
    The package ships two commands: <code>create-stackr</code> generates a new
    project, and <code>stackr</code> operates <em>inside</em> an existing one
    (adding services, adding entities, checking for drift). See the
    <a href="/docs/cli-reference">CLI Reference</a>.
</Callout>

## Next steps

Head to the [Quick Start](/docs/quick-start) to install dependencies, boot the
stack, and run the tests — or jump to [Configuration](/docs/configuration) to walk
